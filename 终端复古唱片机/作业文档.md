# 终端复古唱片机 — Linux 工具综合应用

> **核心工具**: `jp2a` (图片转 ASCII 艺术), `ffplay` / `sox` (音频播放)
> **辅助工具**: Shell 脚本, `tput` (终端控制)
> **场景**: 在终端中打造一台"复古唱片机"——播放音乐、显示同步歌词、呈现彩色 ASCII 像素封面

---

## 一、设计

### 1.1 场景构思

图形化音乐播放器比比皆是，但如果在纯文本终端里——没有 GUI、没有窗口——也能放歌，而且还能看到唱片封面变成的**彩色像素画**、歌词逐句浮现，那会是怎样一种体验？

这就是 **"终端复古唱片机"** 的灵感来源。它模拟了一台老式黑胶唱片机的工作方式：

1. 收集"唱片"（每首歌封装为一个目录，内含音频、歌词、封面）
2. 打开终端，敲下 `./music.sh`
3. 终端清空，彩色 ASCII 像素封面缓缓铺开
4. 音乐响起，歌词逐句高亮
5. 播完自动切换到下一首歌，全部播完安静退出
6. 支持循环播放模式

这是一个将 Linux 哲学——"一切皆文件、小工具协作"发挥到极致的场景。

### 1.2 工具选择理由

| 角色 | 工具 | 为什么选它 |
|------|------|-----------|
| **核心 — 图像转换** | `jp2a` | 将 JPEG/PNG 图片转为终端彩色输出。支持 24-bit 真彩色 + 字符背景填充，形成像素画效果 |
| **核心 — 音频播放** | `ffplay`（默认）/ `sox` | ffplay 提供高品质音频播放；sox 是 Linux 音频"瑞士军刀"，支持音效处理和格式转换 |
| 辅助 — 终端控制 | `tput` + ANSI | 光标定位、真彩色、清屏——在终端里画 UI 的基础能力 |
| 串联 — 主控逻辑 | Shell 脚本 | 将上述工具像搭积木一样组合，实现定时、同步、交互 |

**为什么用 `jp2a` 而不是 Python/Pillow？**

因为 `jp2a` 是 Linux 原生工具，本身就支持 `--colors --color-depth=24 --fill` 真彩色输出。配合 `--chars='##'` 参数，每个像素用不同颜色的 `#` 表示，形成独特的终端像素画风格——纯 Linux 工具栈，不引入外部依赖。

**音频为什么提供 ffplay 和 sox 双后端？**

`ffplay`（FFmpeg 套件）处理 WSL 音频桥接更稳定，音质更好。`sox` 是 Linux 经典音频工具，支持 `rate -v -L`（线性相位最高质量重采样）、`gain -n`（自动归一化）、`dither -S`（噪声整形）等专业音频处理参数。用户可通过 `-e` 切换。

### 1.3 "唱片封装"设计

每首歌作为一个独立的"唱片包"，自包含所有素材：

```
linux/
├── music.sh              # 主程序入口
├── songs/                # 唱片架
│   ├── robin/            # 一张"唱片"
│   │   ├── audio.mp3     #   ⬛ 音频文件（任意格式）
│   │   ├── lyrics.lrc    #   ⬛ 歌词（可选）
│   │   └── cover.png     #   ⬛ 封面图（可选）
│   └── swing/            # 另一张"唱片"
│       ├── audio.mp3
│       ├── lyrics.lrc
│       └── cover.png
└── cache/                # 像素画缓存（自动生成）
```

- 新建文件夹 = "发售新唱片"
- 放入音频 + 歌词 + 封面 = 封装完毕
- 脚本自动扫描 `songs/` 目录，构建播放列表

## 二、实现

### 2.1 环境与依赖

- **操作系统**: Linux（Ubuntu / Debian / Arch 等），或 WSL
- **依赖安装**:
  ```bash
  sudo apt install sox jp2a ffmpeg pulseaudio-utils
  ```

### 2.2 关键技术点

#### 2.2.1 音频播放（ffplay / sox 双后端）

默认使用 `ffplay` 播放，获得最佳音质：

```bash
ffplay -nodisp -autoexit -loglevel quiet "$audio" &>/dev/null &
```

切换回 `sox`（`-e sox`）时，使用最高质量参数：

```bash
sox "$audio" -d rate -v -L 44100 gain -n -1 dither -S 2>/dev/null &
```

| 参数 | 含义 |
|------|------|
| `rate -v -L 44100` | 线性相位最高质量重采样 |
| `gain -n -1` | 自动归一化到 -1dBFS，防止削波 |
| `dither -S` | 噪声整形抖动，减少量化失真 |

通过 `kill -0 $PID` 检测进程存活判断播放进度，`SIGSTOP`/`SIGCONT` 实现暂停/恢复。

#### 2.2.2 jp2a 真彩色像素画

`jp2a` 不仅支持灰度 ASCII，还支持 24-bit 真彩色输出：

```bash
jp2a --colors --color-depth=24 --fill --chars='##' --width=80 cover.png
```

| 参数 | 作用 |
|------|------|
| `--colors` | 启用真彩色模式 |
| `--color-depth=24` | 1670 万色 |
| `--fill` | 填充字符背景色，形成实心色块 |
| `--chars='##'` | 全部使用 `#` 字符，彩色像素画效果 |

每个像素 = 一个带独立 RGB 背景色的 `#` 字符。结果缓存在 `cache/` 目录，避免重复计算。

#### 2.2.3 LRC 歌词解析与元数据过滤

LRC 文件有时会混入制作信息（作曲、作词、录音棚等），这些不是歌词但会污染显示。解析时自动过滤：

```bash
if [[ "$text" =~ (作曲|作词|编曲|制作人|人声|电吉他|和声|录音棚|录音师|混音师|母带|出品) ]]; then
    continue  # 跳过制作信息行
fi
```

同时过滤掉 `[00:00.00]` 处的歌名/艺术家信息行。

#### 2.2.4 中文视觉宽度截断

终端中中文字符占 2 列，英文占 1 列。直接用 `${var:0:N}` 字节截断会导致中文文本实际宽度远超面板。解决方案：`trunc_visual()` 函数按视觉宽度截断。

```bash
trunc_visual() {
    local str="$1" maxw="$2" out="" w=0
    local i ch byte
    for ((i=0; i<${#str}; i++)); do
        ch="${str:$i:1}"
        byte=$(printf "%d" "'$ch")
        if [[ $byte -gt 127 ]]; then
            w=$((w + 2))   # 非 ASCII → 宽字符
        else
            w=$((w + 1))
        fi
        [[ $w -gt $maxw ]] && { out+="…"; break; }
        out+="$ch"
    done
    echo "$out"
}
```

所有非 ASCII 字节（>127）视为宽字符（2 列），ASCII 为 1 列，精确控制面板内文本不溢出。

#### 2.2.5 增量渲染（防闪烁）

第一版每帧重绘整个面板（~11 次 `tput` + `printf`），终端跟不上导致闪烁。优化方案：拆分静态/动态绘制。

- `draw_panel_frame()` — 歌曲开始时调用**一次**，绘制背景、标题、分割线
- `draw_panel_dynamic()` — 每帧调用，只更新变化的 3-4 行（歌词仅在内容改变时更新、进度条、底部信息）

终端 I/O 减少约 70%，面板稳定不闪。

#### 2.2.6 非阻塞按键监听

```bash
read_key() {
    read -r -t 0.08 -n 1 key 2>/dev/null || { echo ""; return; }
    if [[ "$key" == $'\e' ]]; then
        read -r -t 0.01 -n 1 k2
        [[ "$k2" == '[' ]] && read -r -t 0.01 -n 1 k3
    else
        echo "$key"
    fi
}
```

支持 Space（暂停）、q/Esc（退出）。

#### 2.2.7 播放模式控制

```bash
while true; do
    play_song "$name" "$CURRENT" "$total"
    # 自然播完后根据 mode 决定前进或退出
    if [[ "$MODE" == "loop" ]]; then
        CURRENT=$(( (CURRENT + 1) % ${#PLAYLIST[@]} ))
    else
        CURRENT=$(( CURRENT + 1 ))
        [[ $CURRENT -ge ${#PLAYLIST[@]} ]] && break
    fi
done
```

#### 2.2.8 暂停/恢复

利用 Linux 进程信号 `SIGSTOP` / `SIGCONT`：

```bash
kill -STOP "$SOX_PID"   # 暂停（冻结进程）
kill -CONT "$SOX_PID"   # 继续（解冻进程）
```

暂停期间累计的暂停时长从总耗时中扣除，保证歌词同步不受影响。

### 2.3 完整代码 (music.sh)

> 详见 `music.sh` 文件，约 560 行 Shell 脚本。

核心函数清单：

| 函数 | 功能 |
|------|------|
| `scan_songs()` | 扫描 `songs/` 目录，构建播放列表 |
| `load_ascii()` | 调用 jp2a 生成/加载缓存彩色像素画 |
| `parse_lrc()` | 解析 `.lrc` 歌词文件，过滤制作元数据 |
| `trunc_visual()` | 按视觉宽度截断字符串（中文 2 列、英文 1 列） |
| `play_song()` | 播放单首歌曲的主循环（音频+歌词+按键） |
| `draw_panel_frame()` | 绘制面板静态部分（标题、背景，仅一次） |
| `draw_panel_dynamic()` | 增量更新面板动态部分（歌词、进度条） |
| `read_key()` | 非阻塞读取键盘输入 |
| `main()` | 参数解析、模式控制、歌曲调度 |

### 2.4 使用方式

```bash
# 基本用法
./music.sh                 # 顺序播放 songs/ 下所有歌曲
./music.sh -m loop         # 循环播放
./music.sh -s robin        # 只播放 robin 这首歌
./music.sh -s robin -m loop  # 循环播放指定歌曲
./music.sh -e sox          # 使用 sox 音频后端
./music.sh -l              # 列出所有可用歌曲
./music.sh -h              # 查看帮助

# 运行时按键
Space     暂停/继续
q / Esc   退出
```

### 3.3 技术收获

1. **jp2a 不止于灰度**：通过 `--colors --fill` 参数，jp2a 可以输出 24-bit 真彩色终端图像。配合 `--chars` 控制字符集，可以实现终端像素画——这展示了 Linux 工具"深挖参数即解锁新能力"的哲学。

2. **sox 音频处理专业化**：`rate -v -L`（线性相位）、`gain -n`（归一化）、`dither -S`（噪声整形）等参数体现了专业音频工具链的能力。这些不是简单的"播放"功能，而是完整的音频后处理管线。

3. **终端即画布**：通过 ANSI 转义序列和 `tput`，一个纯文本终端可以做到 24-bit 真彩色、光标定位、清屏、隐藏光标——足以构建一个完整的 TUI（文本用户界面）。

4. **增量渲染优化**：将渲染拆分为静态帧 + 动态更新，每帧只重绘变化的行。这是游戏/UI 开发的经典模式，在 Shell 脚本中同样适用。

5. **信号机制的实用化**：`SIGSTOP`/`SIGCONT` 用于暂停/恢复音频播放，`SIGINT`/`SIGTERM` 的 trap 用于优雅退出，这些是 Linux 进程管理的典型应用。

6. **国际化文本处理**：终端中 ASCII 占 1 列、中文占 2 列。简单的 `${var:0:N}` 截断对混合文本无效。通过字节值判断宽字符（>127 即非 ASCII），实现了精确的视觉宽度控制。

7. **脚本组合力**：一个 560 行的 Shell 脚本，通过组合 `jp2a`、`ffplay`/`sox`、`tput`、`find`、`kill` 等独立 Linux 工具，实现了一个完整的彩色终端音乐播放器。这就是 Unix 哲学的威力——**小工具，大组合**。
